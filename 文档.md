### 3. JwtUtil.java - JWT工具类

**关键代码**:
```java
public String generateToken(String userId, String username) {
    Map<String, Object> claims = new HashMap<>();
    claims.put("userId", userId);
    claims.put("username", username);
    
    Date now = new Date();
    Date expiryDate = new Date(now.getTime() + expiration); // 24小时
    
    return Jwts.builder()
        .claims(claims)
        .subject(username)
        .issuedAt(now)
        .expiration(expiryDate)
        .signWith(getSigningKey()) // HMAC-SHA256
        .compact();
}
```

export const AuthProvider: React.FC<AuthProviderProps> = ({ children }) => {
  const [user, setUser] = useState<User | null>(null)
  const [loading, setLoading] = useState(true)

  // 初始化：从localStorage读取token并验证
  useEffect(() => {
    const initAuth = async () => {
      const token = localStorage.getItem('token')
      if (token) {
        try {
          const response = await authService.getCurrentUser()
          setUser(response.data)
        } catch (error) {
          localStorage.removeItem('token')
        }
      }
      setLoading(false)
    }
    initAuth()
  }, [])

  const login = async (username: string, password: string) => {
    const response = await authService.login(username, password)
    if (response.data?.user) {
      setUser(response.data.user)
    }
  }

  const logout = () => {
    authService.logout()
    setUser(null)
  }

  return (
    <AuthContext.Provider value={{ user, loading, login, loginByCode, register, logout, updateUser }}>
      {children}
    </AuthContext.Provider>
  )
}
```

## 核心流程

### 注册流程
```
1. 用户填写注册信息（手机号、验证码、密码等）
   ↓
2. 前端调用 authService.register()
   ↓
3. 后端 AuthController.register() 接收请求
   ↓
4. 验证验证码（VerificationCodeService.verifyCode）
   ↓
5. AuthService.register() 处理业务逻辑：
   - 用户名处理（自动生成或使用提供的）
   - 检查用户名是否已存在
   - BCrypt加密密码
   - 创建用户记录
   - 自动绑定支付宝账号（手机号）
   ↓
6. 生成JWT token（JwtUtil.generateToken）
   ↓
7. 创建用户会话（SessionService.createSession）
   ↓
8. 返回用户信息和token
   ↓
9. 前端保存token到localStorage
   ↓
10. 更新AuthContext中的用户状态
```

### 密码登录流程
```

## 技术要点

### 密码加密
- 使用 **BCrypt** 算法加密密码
- Spring Security的 `BCryptPasswordEncoder`
- 每次加密结果不同，但可以正确验证

### JWT Token
- 算法：HMAC-SHA256
- 有效期：24小时（86400000毫秒）
- 包含信息：userId、username
- 存储位置：前端localStorage

### 会话管理
- 使用 `SessionService` 管理用户会话
- 支持异地登录挤掉其他会话
- token失效时自动登出

### 验证码机制
- 发送验证码：`POST /auth/send-code`
- 验证验证码：`POST /auth/verify-code`
- 注册和验证码登录都需要验证码
- 验证码有时效性（默认5分钟）

### 用户名自动生成规则
- 如果用户名为空，使用手机号后4位
- 如果后4位已存在，自动添加数字后缀（1, 2, 3...）
- 最多尝试到9999

### 支付宝账号自动绑定
- 注册时自动将手机号设置为支付宝账号
- 登录时如果支付宝账号为空，自动绑定手机号
- 更新手机号时，如果支付宝账号与旧手机号相同，自动更新为新手机号

### 核心流程

#### 智能定价流程
```
1. 用户在发布商品页面填写商品信息（类别、成色、标题、描述、原价等）
   ↓
2. 点击"一键智能定价"按钮
   ↓
3. 前端调用 productService.getPriceSuggestion(request)
   ↓
4. 后端 ProductController.getPriceSuggestion() 接收请求
   ↓
5. 搜索相似商品（多策略匹配）：
   - 策略1：完全匹配（相同类别和成色）
   - 策略2：只匹配类别（不限制成色）
   - 策略3：只匹配主类别
   ↓
6. 如果找到相似商品：
   - 提取相似商品价格
   - 调用DeepSeek AI进行智能定价
   - 构建包含商品信息和相似价格的提示词
   - 调用DeepSeek API（model: deepseek-chat）
   - 解析AI返回的价格数字
   ↓
7. 如果AI失败或未找到相似商品：
   - 使用传统算法兜底（基于成色和原价计算）
   ↓
8. 计算价格区间、置信度、影响因素权重
   ↓
9. 返回价格建议：
   - recommendedPrice（推荐价格）
   - minPrice/maxPrice（价格区间）
   - confidence（置信度）
   - factors（影响因素权重）
   - similarProducts（相似商品列表）
   - algorithm（使用的算法：deepseek_ai）
   ↓
10. 前端自动填入推荐价格到表单
   ↓
11. 显示定价结果和说明
```

### 技术要点

#### DeepSeek模型配置
- **模型**: `deepseek-chat`（使用配置的ai.api.model）
- **API地址**: `https://api.deepseek.com/chat/completions`
- **Temperature**: 0.3（降低温度以获得更稳定的定价结果）
- **Max Tokens**: 100（只需返回数字，不需要长文本）

#### 相似商品搜索策略
- **策略1**: 完全匹配（相同类别和成色）
- **策略2**: 只匹配类别（放宽成色限制）
- **策略3**: 只匹配主类别（如果类别是"主类/子类"格式）
- 最多返回20个相似商品
- 过滤已删除和未发布的商品

#### 定价算法
- **AI定价**（优先）：
  - 使用DeepSeek分析商品信息、成色、原价、相似商品价格
  - 综合考虑市场行情、商品特性、供需关系
  - 置信度：0.85（基础）+ 相似商品数量 * 0.02
- **传统算法**（兜底）：
  - 基于成色和原价计算（全新90%、近新80%、良好70%、一般60%、较差50%）
  - 使用中位数和平均价的加权平均
  - 置信度：0.6

#### 价格区间计算
- **最低价**: 推荐价格 * 0.8
- **最高价**: 推荐价格 * 1.2
- **推荐价格**: AI返回或传统算法计算的价格（保留2位小数）

#### 影响因素权重
- `condition`: 0.3（商品成色）
- `category`: 0.3（商品类别）
- `marketTrend`: 0.2（市场趋势）
- `aiAnalysis`: 0.2（AI分析）

#### 提示词设计
- **System Prompt**: "你是一个专业的二手商品定价专家。请根据用户提供的商品信息，给出合理的定价建议。只返回数字，不要包含任何其他文字。"
- **User Prompt**: 包含商品标题、描述、类别、成色、原价、相似商品价格参考、平均价格等
- **输出要求**: 只返回数字，不包含任何其他文字或符号

#### 错误处理
- API未配置时使用传统算法
- API调用失败时使用传统算法兜底
- AI返回无效值时使用传统算法
- 从AI回复中提取数字（去除所有非数字字符）
- 验证价格有效性（> 0）

  minPrice: number          // 最低价
  maxPrice: number          // 最高价
  confidence: number        // 置信度（0-1）
  factors: {                // 影响因素权重
    condition: number
    category: number
    marketTrend: number
    aiAnalysis: number
  }
  similarProducts: Product[] // 相似商品列表（最多5个）

#### 2. location.ts - 位置转换工具
**路径**: `frontend/src/utils/location.ts`

**核心函数**:
```typescript
/**
 * 将拼音格式的 location 转换为中文显示
 * 例如: "guangdong / shenzhen / sust" -> "广东省 / 深圳市 / 深圳大学"
 */
export const convertLocationToChinese = (location: string | undefined): string => {
  if (!location) return ''
  
  // 如果已经是中文（包含中文字符），直接返回
  if (/[\u4e00-\u9fa5]/.test(location)) {
    return location
  }
  
  // 分割 location 字符串
  const parts = location.split(' / ').map(part => part.trim()).filter(Boolean)
  if (parts.length === 0) return location
  
  const labels: string[] = []
  
  // 三级结构：省份 -> 城市 -> 学校
  if (parts.length >= 1) {
    // 查找省份
    const province = schoolData.find(p => p.value === parts[0])
    if (province) {
      labels.push(province.label)
      
      // 如果有城市层级
      if (parts.length >= 2 && province.children) {
        const city = province.children.find(c => c.value === parts[1])
        if (city) {
          labels.push(city.label)
          
          // 如果有学校层级
          if (parts.length >= 3 && city.children) {
            const school = city.children.find(s => s.value === parts[2])
            if (school) {
              labels.push(school.label)
            }
          }
        }
      } else if (parts.length >= 2 && province.children) {
        // 省份直接包含学校（如北京市）
        const school = province.children.find(s => s.value === parts[1])
        if (school) {
          labels.push(school.label)
        }
      }
    }
  }
  
  return labels.length > 0 ? labels.join(' / ') : location
}
```
