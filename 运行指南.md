# 项目运行指南

## ⚠️ 重要提示

**如果遇到编译错误 `java.lang.ExceptionInInitializerError: com.sun.tools.javac.code.TypeTag :: UNKNOWN`，这是因为 Java 版本不兼容。**

**项目需要 Java 17，但系统可能安装了 Java 24 或其他版本。请先安装 Java 17，详见 `Java安装指南.md`**

## 环境要求

### 后端环境
- **Java**: JDK 17（必须，不支持 Java 24 或其他过新版本）
- **Maven**: 3.6+ 
- **MySQL**: 5.7+ 或 8.0+
- **IDE**: IntelliJ IDEA / Eclipse / VS Code（推荐）

### 前端环境
- **Node.js**: >= 16.0.0
- **npm**: >= 7.0.0 或 **yarn**: >= 1.22.0

## 一、数据库准备

### 1. 创建数据库

```sql
CREATE DATABASE campus_trading CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 配置数据库连接

编辑 `backend/src/main/resources/application.yml`，修改数据库配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_trading?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root        # 修改为你的MySQL用户名
    password: 123456      # 修改为你的MySQL密码
```

**注意**：项目配置了自动初始化数据库表结构，首次启动会自动执行 `schema.sql` 和 `data.sql`。

## 二、后端运行

### 方式一：使用IDE运行（推荐）

1. **打开项目**
   - 使用 IntelliJ IDEA 打开 `backend` 目录
   - 等待 Maven 自动下载依赖

2. **找到主类**
   - 路径：`src/main/java/com/campus/trading/Application.java` 或类似的主类

3. **运行主类**
   - 右键点击主类 → `Run 'Application'`
   - 或使用快捷键运行

4. **验证启动**
   - 看到日志显示 `Started Application in X.XXX seconds`
   - 访问 `http://localhost:8080/api` 测试（应该返回404或API响应）

### 方式二：使用Maven命令运行

```bash
# 进入后端目录
cd backend

# 编译项目
mvn clean compile

# 运行项目
mvn spring-boot:run
```

### 方式三：打包后运行

```bash
# 进入后端目录
cd backend

# 打包项目
mvn clean package

# 运行jar包
java -jar target/second-hand-trading-platform-1.0.0.jar
```

### 后端配置说明

- **端口**: 8080
- **API路径**: `/api`（所有接口都在此路径下）
- **完整地址**: `http://localhost:8080/api`

**重要配置项**（`application.yml`）：
- 数据库连接信息
- JWT密钥和过期时间
- 文件上传路径
- 支付宝配置（沙箱环境）
- AI API配置（DeepSeek）

## 三、前端运行

### 1. 安装依赖

```bash
# 进入前端目录
cd frontend

# 安装依赖（使用npm）
npm install

# 或使用yarn
yarn install
```

### 2. 环境配置（可选）

创建 `.env` 文件（如果不存在）：

```env
VITE_API_BASE_URL=http://localhost:8080/api
```

**注意**：如果不创建 `.env` 文件，默认使用 `/api` 作为后端地址（相对路径）。

### 3. 启动开发服务器

```bash
# 使用npm
npm run dev

# 或使用yarn
yarn dev
```

### 4. 访问应用

浏览器打开：`http://localhost:3000`

### 前端其他命令

```bash
# 构建生产版本
npm run build

# 预览生产构建
npm run preview

# 代码检查
npm run lint
```

## 四、完整启动流程

### 步骤1：启动MySQL数据库

**重要**：必须先启动 MySQL 服务，否则后端会报 `Connection refused` 错误。

确保MySQL服务正在运行：

```bash
# macOS (使用 Homebrew)
brew services start mysql

# 或使用 mysql.server
mysql.server start

# 验证 MySQL 是否运行
lsof -i :3306
# 如果看到输出，说明 MySQL 正在运行

# 如果 MySQL 未安装，先安装
brew install mysql
brew services start mysql
```

**创建数据库**（如果未创建）：
```bash
# 连接 MySQL
mysql -u root -p

# 在 MySQL 命令行中执行
CREATE DATABASE campus_trading CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
exit;
```

### 步骤2：启动后端服务

```bash
cd backend
mvn spring-boot:run
```

等待看到：
```
Started Application in X.XXX seconds
```

### 步骤3：启动前端服务

打开新的终端窗口：

```bash
cd frontend
npm install  # 首次运行需要
npm run dev
```

等待看到：
```
VITE v5.x.x  ready in XXX ms
➜  Local:   http://localhost:3000/
```

### 步骤4：访问应用

浏览器打开：`http://localhost:3000`

## 五、常见问题

### 1. 后端启动失败

**问题：编译错误 `java.lang.ExceptionInInitializerError: com.sun.tools.javac.code.TypeTag :: UNKNOWN`**
```
原因：Java版本不兼容，项目需要Java 17，但系统可能使用了Java 24或其他版本

解决方案：
1. 检查Java版本：java -version 和 mvn -version
2. 安装Java 17（详见 Java安装指南.md）：
   brew install openjdk@17
3. 配置JAVA_HOME环境变量指向Java 17
4. 重新编译：mvn clean compile
```

**问题：数据库连接失败 - Connection refused**
```
错误信息：
  Caused by: java.net.ConnectException: Connection refused
  com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure

原因：MySQL 服务未启动

解决方案：
1. 启动 MySQL 服务：
   brew services start mysql
   或
   mysql.server start

2. 验证 MySQL 是否运行：
   lsof -i :3306
   如果看到输出，说明 MySQL 正在运行

3. 检查数据库配置（application.yml）：
   - url: jdbc:mysql://localhost:3306/campus_trading
   - username: root（你的 MySQL 用户名）
   - password: 123456（你的 MySQL 密码）

4. 确认数据库已创建：
   mysql -u root -p -e "SHOW DATABASES LIKE 'campus_trading';"
   如果不存在，创建数据库：
   mysql -u root -p -e "CREATE DATABASE campus_trading CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

**问题：端口被占用**
```
解决方案：
- 修改application.yml中的server.port
- 或关闭占用8080端口的程序
```

**问题：Maven依赖下载失败**
```
解决方案：
- 检查网络连接
- 配置Maven镜像源（阿里云镜像）
- 清理并重新下载：mvn clean install
```

### 2. 前端启动失败

**问题：Node版本不兼容**
```
解决方案：
- 使用nvm切换Node版本：nvm use 16
- 或升级Node到16+版本
```

**问题：依赖安装失败**
```
解决方案：
- 删除node_modules和package-lock.json
- 重新安装：npm install
- 或使用yarn：yarn install
```

**问题：后端API请求失败**
```
解决方案：
- 确认后端服务已启动
- 检查.env文件中的VITE_API_BASE_URL配置
- 检查浏览器控制台的网络请求
```

### 3. 数据库问题

**问题：表结构未自动创建**
```
解决方案：
- 检查application.yml中的初始化配置
- 手动执行backend/src/main/resources/db/schema.sql
- 检查数据库用户权限
```

**问题：字符编码问题**
```
解决方案：
- 确保数据库使用utf8mb4字符集
- 检查连接URL中的characterEncoding参数
```

## 八、快速验证

启动成功后，可以通过以下方式验证：

1. **后端健康检查**
   ```bash
   curl http://localhost:8080/api/auth/me
   # 应该返回401（未授权）或用户信息
   ```

2. **前端页面**
   - 访问 `http://localhost:3000`
   - 应该能看到首页

3. **API测试**
   - 注册接口：`POST http://localhost:8080/api/auth/register`
   - 登录接口：`POST http://localhost:8080/api/auth/login`
